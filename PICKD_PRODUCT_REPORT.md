## 1. 서비스 개요

**pickD**는 AI 기반 뷰티 분석과 커뮤니티를 결합한 개인 맞춤형 뷰티 플랫폼입니다.

| 항목 | 내용 |
|------|------|
| 플랫폼 | iOS Native + Web (Vercel 배포) |
| 프레임워크 | Expo (React Native) |
| 백엔드 | Supabase (PostgreSQL + 실시간 DB + 인증) |
| 지도 | Kakao Maps API |
| 결제 | 포인트 시스템 (베타) / Toss Payments (예정) |
| 타겟 사용자 | 뷰티·피부 관리에 관심 있는 18~50세 여성 |

### 앱 구성
- **탭 5개**: 홈 / 커뮤니티 / 검색 / 비교 / 마이페이지
- **전체 화면 수**: 30개 이상
- **핵심 기능**: AI 피부 진단 / 시술·기기 탐색 / 비교 추천 / 커뮤니티 / 피부과 지도

## 2. 핵심 가치 제안

### 기존 문제
- 피부 시술·기기 정보가 분산되어 있어 비교가 어렵다
- 내 피부타입·얼굴형에 맞는 시술이 무엇인지 모른다
- 실제 사용 후기를 신뢰할 수 없다
- 주변 피부과를 찾아다니기 번거롭다

### pickD 해결책

| 문제 | pickD 솔루션 |
|------|-------------|
| 분산된 정보 | 시술·기기 DB + 비교 기능으로 한 곳에서 해결 |
| 내 맞춤 정보 부재 | Baumann 피부타입 × 얼굴형 AI 진단 |
| 후기 신뢰 부족 | 스팸 감지 + 리뷰 신뢰도 점수 |
| 병원 탐색 불편 | Kakao Maps 실시간 주변 피부과 검색 |

## 3. 사용자 플로우 (고객 여정)

### 3-1. 신규 사용자 온보딩 플로우

```
[앱 설치]
    ↓
[회원가입]
  · 이메일 + 비밀번호
  · 닉네임, 나이, 성별, 피부타입, 고민
  · 회원가입 완료 → 1,000pt 자동 지급
    ↓
[홈 화면]
  · 🎉 1,000pt 지급 팝업 표시
  · AI 피부 분석 배너 노출
    ↓
[얼굴형 분석] (5문항)           [피부타입 분석] (8문항)
  · 이마·광대·턱선·비율 등           · Baumann 4축 (D/O, S/R, P/N, W/T)
  · 6가지 얼굴형 결과               · 16가지 조합 코드 (ex. DRNT)
  · 맞춤 시술·스타일링 가이드         · 6대 피부 지표 점수 (0~100)
    ↓
[마이페이지 > 보관함]
  · 얼굴형 결과 저장 & 열람
  · 피부타입 결과 저장 & 열람
    ↓
[맞춤 피부 보고서 구매] (990pt)
  · 시술 로드맵 + 스킨케어 루틴 + 스타일링 가이드
```

### 3-2. 재방문 사용자 일상 플로우

```
[매일 출석 체크] → +10pt
    ↓
[홈] → 피부 관심사별 시술·기기 페어링 탐색
    ↓
[검색] → 시술/기기 검색 + 카테고리 필터
    ↓
[비교함 추가] (최대 3개)
    ↓
[AI 추천 받기]
  · 점수 기반 최적 항목 선정
  · 내 피부타입·얼굴형·고민 반영
    ↓
[커뮤니티] → 후기/질문/퀴즈 참여 → +10pt
    ↓
[주변 피부과 찾기] → Kakao 지도
```

### 3-3. 포인트 사용 플로우

```
포인트 적립
  └─ 가입 +1,000 / 출석 +10 / 첫글 +500 / 초대 +500
     댓글·투표·공유 각 +10 / 7일 연속 출석 +100
         ↓
포인트 사용
  └─ 맞춤 피부 보고서 -990pt
     퀴즈 생성 -100pt
         ↓
미션 페이지에서 현재 미션 상태 & 포인트 내역 확인
```

## 4. 화면별 상세 기능

### 4-1. 홈 (Home)

**목적**: 개인화된 뷰티 추천 대시보드

| 섹션 | 내용 |
|------|------|
| 상단 헤더 | 사용자 아바타·닉네임·포인트·미션 배지 |
| 피부 프로필 배너 | 등록된 피부타입·고민 표시 → 프로필 수정 이동 |
| AI 보고서 CTA | 990pt 프리미엄 보고서 구매 유도 배너 |
| 관심사별 페어링 | 리프팅·미백·수분·주름 고민별 시술+기기 묶음 카드 |
| 인기 시술 TOP 6 | 평점순 정렬 / 이름·가격범위·평점·리뷰수 표시 |
| 인기 기기 TOP 6 | 평점순 정렬 / 이름·브랜드·가격·평점 표시 |
| 주변 피부과 지도 | Kakao Maps 미니맵 (웹) / 배너 CTA (앱) |

**데이터 제공 정보**:
- 시술 평균 가격 범위 (최저~최고)
- 평점 (0~5점)
- 리뷰 수
- 관심사 카테고리별 매칭 시술·기기

---

### 4-2. 검색 (Search)

**목적**: 시술·기기 탐색 및 비교함 추가

| 기능 | 상세 |
|------|------|
| 탭 전환 | 시술 / 기기 |
| 검색창 | 실시간 검색 (300ms 디바운스) + 최근 검색어 10개 저장 |
| 카테고리 필터 (시술) | 전체 / 리프팅 / 보톡스 / 필러 / 레이저 / 스킨케어 |
| 카테고리 필터 (기기) | 전체 / 리프팅 / 모공·피지(RF) / 미백·잡티(LED) / 수분·보습(초음파) / 제모 |
| 결과 카드 | 이미지·이름·가격·평점·리뷰수 + +비교 버튼 |
| 지도 연결 | 시술 탭에서 "주변 피부과" 버튼 → 선택 카테고리로 지도 필터 |
| 비교 추가 | 체크마크로 추가/제거 표시, 최대 3개 |

---

### 4-3. 비교 (Compare)

**목적**: 2~3개 항목 나란히 비교 + AI 추천

| 기능 | 상세 |
|------|------|
| 비교 카드 | 썸네일·이름·가격·평점 표시, X 버튼으로 제거 |
| 빈 슬롯 | + 아이콘 → 검색으로 이동 |
| AI 추천 (2개 이상) | 1.8초 로딩 → 최적 항목 선정 + 이유 + 점수 |
| 비용 계산기 | 시술 1회 비용 vs 기기 52회 사용 비교 |
| 상세 비교표 | 평점·리뷰·가격·카테고리 + 시술별/기기별 특수 항목 |

**AI 추천 점수 산출 기준**:

| 항목 | 배점 |
|------|------|
| 피부타입 매칭 | 최대 3점 |
| 고민 매칭 (2점 × 항목 수) | 최대 6점 |
| 얼굴형 적합도 | 최대 2점 |
| 사용자 평점 (평점 × 0.8) | 최대 4점 |
| 리뷰 신뢰도 (200개 이상 = 1점) | 최대 1점 |
| **총점** | **최대 16점** |

**시술 전용 비교 항목**: 통증 강도 / 다운타임 / 지속 기간 / 권장 세션수 / 부작용 / 금기 사항

**기기 전용 비교 항목**: 사용 빈도 / 효과 확인 기간 / 부작용 / 금기 사항

### 4-6. 얼굴형 분석 (Face Analysis)

**목적**: 5문항 설문으로 얼굴형 판별 + 맞춤 가이드 제공

**설문 구성**:

| 번호 | 질문 | 선택지 |
|------|------|--------|
| Q1 | 이마 너비 | 좁음 / 중간 / 넓음 / 매우 넓음 |
| Q2 | 광대 너비 | 좁음 / 중간 / 넓음 / 가장 두드러짐 |
| Q3 | 턱선 형태 | 갸름 / 둥글고 짧음 / 각짐 / 매우 갸름 |
| Q4 | 가로:세로 비율 | 정사각형 / 약간 갸름 / 이마 넓음 / 세로 긺 |
| Q5 | 가장 넓은 부분 | 광대 / 비슷함 / 이마 / 전체 균형 |

**결과 얼굴형 6종**:

| 얼굴형 | 설명 | 핵심 관리 방향 |
|--------|------|--------------|
| 계란형 🥚 | 황금 기준형 | 노화 예방·볼륨 유지 |
| 둥근형 ⭕ | 볼살·짧은 턱선 | 세로 길이감·슬리밍 |
| 사각형 ⬜ | 강한 각진 턱선 | 교근 축소·리프팅 |
| 하트형 | 이마 넓고 턱 좁음 | 이마 볼륨·하안면 강조 |
| 긴형 | 세로가 길고 좁음 | 가로 볼륨·여백 줄이기 |
| 다이아몬드형 | 광대 돌출 | 광대 축소·이마·턱 보완 |

**결과 제공 정보**:
- 💡 보완 포인트 3개
- 💉 추천 시술 TOP 3 (기간·다운타임·예상 비용)
- 🏠 추천 홈케어 디바이스 + 사용 빈도
- ✂️ 스타일링 가이드 (헤어·메이크업·피해야 할 스타일)
- 📅 연령대별 관리 코멘트 (20대 / 30대 초 / 30대 후반 / 40대 이상)

### 4-7. 피부타입 분석 (Skin Analysis)

**목적**: 8문항 Baumann 분류법으로 피부타입 판별 + 6대 지표 분석

**Baumann 4축 분류**:

| 축 | 의미 | 선택지 |
|----|------|--------|
| D/O | 유분·수분 | Dry (건성) / Oily (지성) |
| S/R | 민감도 | Sensitive (민감성) / Resistant (저항성) |
| P/N | 색소침착 | Pigmented (색소성) / Non-pigmented (비색소) |
| W/T | 탄력·주름 | Wrinkle-prone (주름성) / Tight (탄력) |

→ **총 16가지 조합** (ex. DRNT, OSPW 등)

**설문 구성 (총 8문항, 축별 2문항)**:

| 축 | Q | 질문 내용 |
|----|---|----------|
| D/O | Q1 | 세안 후 얼굴이 어떤가요? |
| D/O | Q2 | 낮 12시에 T존 기름기는? |
| S/R | Q3 | 새 제품이나 환경 변화에 피부 반응은? |
| S/R | Q4 | 자외선·오염·운동 후 피부 반응은? |
| P/N | Q5 | 여드름 자국이 남는 편인가요? |
| P/N | Q6 | 햇빛 노출 후 색소 침착은? |
| W/T | Q7 | 현재 피부 탄력감은? |
| W/T | Q8 | 눈가·입가 잔주름은? |

**6대 피부 지표 (0~100점)**:

| 지표 | 설명 | 점수 해석 |
|------|------|----------|
| 모공 | 모공 크기 및 가시성 | 60↑ 관리 필요 / 30~59 보통 / 30↓ 양호 |
| 주름 | 주름·탄력 상태 | 동일 기준 |
| 색소침착 | 기미·잡티 분포 | 동일 기준 |
| UV색소침착 | 자외선성 색소 | 동일 기준 |
| 탄력 | 피부 탄력도 | 동일 기준 |
| 피부톤 | 톤 균일도 | 동일 기준 |

**속건조 감지**: 지성·복합성이라도 건성 점수 일정 이상 시 💧 속건조 태그 추가

**결과 제공 정보**:
- Baumann 코드 배지 + 속건조 여부
- 4축 분류 표 (어떤 축인지 한국어 설명)
- 6대 지표 바 그래프 + 상태
- 피부 특징 5~7개
- 💉 추천 시술 TOP 3 (효과·주기)
- 🏠 추천 디바이스 + 빈도
- 🧴 5단계 스킨케어 루틴 (클렌저·토너·세럼·크림·선크림) + 성분·팁

### 4-8. 분석 리포트 (Analysis Report)

**목적**: 얼굴형 + 피부타입 + 나이를 통합한 맞춤 뷰티 로드맵

**구성 (3파트)**:

**PART 01: 얼굴형 분석**
- 얼굴형 카드 (이모지·이름·비율 설명)
- 보완 포인트
- 추천 시술 TOP 3 (순위·기간·다운타임·비용)
- 홈케어 디바이스
- 스타일링 가이드 (헤어·메이크업·피해야 할 스타일)

**PART 02: 피부타입 분석**
- 피부타입 카드
- Baumann 4축 분류
- 6대 지표 시각화
- 피부 특징
- 추천 시술·디바이스·스킨케어 루틴

**PART 03: 연령대별 코멘트**
- 20대 / 30대 초반 / 30대 후반 / 40대 이상 맞춤 조언
- 얼굴형 + 피부타입 동시 반영

## 5. 포인트 경제 시스템

### 적립 vs 사용

| 구분 | 액션 | 포인트 |
|------|------|--------|
| **적립** | 가입 | +1,000 |
| | 매일 출석 | +10/일 |
| | 7일 연속 출석 | +100/주 |
| | 첫 게시물 | +500 |
| | 친구 초대 | +500/건 |
| | 댓글·투표·공유 | 각 +10/일 |
| **사용** | 맞춤 피부 보고서 | -990 |
| | 퀴즈 생성 | -100 |

### 보안 처리
- 모든 포인트 입출금은 **PostgreSQL RPC 원자적 트랜잭션** 처리
- `FOR UPDATE` 락으로 동시 차감 레이스 컨디션 방지
- 전체 내역 `point_logs` 테이블에 영구 보관

## 6. AI 분석 엔진 상세

### 6-1. 얼굴형 판별 알고리즘

```
각 질문별 선택지 → 얼굴형 점수 매핑
→ 6가지 얼굴형 중 최고 점수 선정
→ 해당 얼굴형의 시술·스타일링 가이드 표시
```

- 점수 테이블: 각 선택지별로 관련 얼굴형에 1~2점 부여
- 동점 시: 배열 우선순위 (계란형 최우선)

### 6-2. Baumann 피부타입 판별

```
8문항 답변 → 4축별 D/O, S/R, P/N, W/T 점수 집계
→ 각 축 우세값 1글자 선택
→ 4글자 코드 생성 (ex. DRNT)
→ 6대 지표 점수 계산 (답변 조합 기반)
→ 속건조 감지 (지성이면서 건성 점수 ≥ 임계값)
```

### 6-3. 비교 AI 추천 알고리즘

```
각 항목별 점수 계산 (최대 16점):
  [피부타입 매칭] → 태그 기반 최대 3점
  [고민 매칭]    → 태그 기반 항목당 2점, 최대 6점
  [얼굴형 적합]  → 카테고리-얼굴형 매핑 최대 2점
  [사용자 평점]  → rating × 0.8, 최대 4점
  [리뷰 신뢰]   → 200개 이상 = 1점
→ 최고 점수 항목이 승자
→ 이유 문구 자동 생성
→ 항목별 점수 시각화 (막대 그래프)
```

## 7. 데이터베이스 구조

### 주요 테이블

| 테이블 | 역할 | 주요 컬럼 |
|--------|------|----------|
| `profiles` | 사용자 프로필 | user_id, nickname, skin_type, face_shape, baumann_code, skin_metrics, skin_dehydration, points, age_group, gender, concerns, avatar_url, face_photo_url |
| `treatments` | 시술 카탈로그 | name, category, description, price_min/max, tags, rating, review_count, pain_level, downtime, duration, sessions, side_effects, contraindications |
| `devices` | 홈케어 기기 | name, brand, category, description, price, tags, rating, review_count, images, coupang_url, usage_freq, results_timeline |
| `posts` | 커뮤니티 게시글 | title, body, category, user_id, likes, comment_count |
| `comments` | 댓글 | post_id, user_id, body |
| `reviews` | 시술·기기 리뷰 | treatment_id/device_id, user_id, rating, body |
| `compare_items` | 비교함 | user_id, item_type, item_id |
| `point_logs` | 포인트 내역 | user_id, amount, reason, created_at |
| `quiz_votes` | 퀴즈 투표 | post_id, user_id, option |
| `favorites` | 찜한 목록 | user_id, item_type, item_id |
| `payments` | 결제 내역 | user_id, amount, status, order_id |

### RLS 보안 정책
- 모든 테이블 RLS(Row Level Security) 활성화
- 본인 데이터만 INSERT·UPDATE·DELETE 가능
- 시술·기기·게시글은 전체 공개 SELECT
